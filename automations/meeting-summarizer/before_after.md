# meeting-summarizer — Before / After

## Before (수동)

1주 1회 30-60분 짜리 팀 미팅 후 한 명이 회의록을 정리한다고 가정.

| 작업 | 평균 소요 |
|---|---|
| Zoom/Teams 자막 다운로드 + 잡음 (타임스탬프, cue 번호) 정리 | 5-10 분 |
| 의제·결정·다음 단계 한 페이지 요약 작성 | 15-25 분 |
| 액션 아이템 표 만들기 (담당자/기한/우선순위 정리) | 10-15 분 |
| 책임자별 후속 메일 1:1 작성 (책임자 N=3-5명 가정, 정중한 어투) | 10-15 분 (메일당 3-5 분) |
| 누락된 액션 한 번 더 훑어보기 (자기-검토) | 5-10 분 |
| **합계** | **45-75 분** |

> 기준: 한국어 30분 미팅, 액션 아이템 5-8 개, 책임자 3-5 명.
> 영어 미팅도 유사 — 다만 한국어 정중한 어투 작성 부담이 빠져서 메일
> 단계가 약간 더 빠름.

## After (자동)

```bash
./frame run meeting-summarizer --in <transcript> "<옵션 컨텍스트>"
```

| 작업 | 평균 소요 |
|---|---|
| `frame run` 호출 (transcript 1 개) | 5-10 초 |
| Claude 가 preprocess → 3 outputs → refute-first auditor 1-2 회 실행 | 60-120 초 (모델 호출 합산) |
| 사용자가 생성된 3 파일 검토 + 미세 수정 | 5-10 분 |
| **합계** | **6-12 분** |

> 모델 호출 시간은 transcript 길이·액션 수에 비례. 30 분 분량 자막
> 기준 1-2 분, 1 시간 분량은 2-4 분 예상.

## 절감 효과

| | 수동 | 자동 | 절감 |
|---|---|---|---|
| 1 회당 시간 | 45-75 분 | 6-12 분 | **~85%** |
| 주 1 회 기준 연간 | 39-65 시간 | 5-10 시간 | 34-55 시간/년 |

## 무엇이 자동화되지 않는가 (정직)

- **검토 단계는 그대로 사람** — refute-first auditor 가 누락 후보를
  찾아 한 번 자동 반영하지만, 최종 confidence 는 사람이 들고 있어야
  함. `audit.md` 의 verdict 가 `no misses found` 여도 발송 전 한 번
  훑는 것을 권장.
- **메일 발송은 수동** — 스킬은 초안 파일만 생성. 실제 발송 (Gmail
  /Outlook/Slack DM) 은 별도 작업.
- **회의 진행 자체** — 자동화 대상 아님. 미팅 자체의 효율 (의제 정리,
  결정 도출) 은 사람의 몫.
- **민감 정보 자동 마스킹 없음** — 회의록에 고객명·계약 금액·미공개
  로드맵이 섞여 있어도 스킬은 그대로 처리. 보안 등급이 높은 미팅은
  사용 전 수동 마스킹 또는 추후 마일스톤 2 (`docs/security.md`) 의
  PII 자동 마스킹 도입 권장.

## 비용 (대략)

- Anthropic API 호출: 30 분 미팅 기준 약 $0.03-0.10 / 회
  (Sonnet 모델 기준, refute-first auditor 1 회 포함)
- 주 1 회 사용 시 연간 약 $1.5-5.2

수동 작업의 사람 시간 가치 (시간당 $25 기준) 대비 100 배 이상의 ROI.
