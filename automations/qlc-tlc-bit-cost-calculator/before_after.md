# qlc-tlc-bit-cost-calculator — Before / After

## Before (수동)

메모리 공정/기획 엔지니어가 QLC 전용 + TLC 전용 분리 생산 vs 콤보 라인
비교를 엑셀로 직접 만든다고 가정.

| 작업 | 평균 소요 |
|---|---|
| 파라미터 정리 (Step 수/투자비/Gross die/수율/Max capa/Wafer 생산량/bit 생산량, QLC·TLC·콤보 3세트) | 15-25 분 |
| 분리 생산 총 bit/총 cost 수식 작성 | 10-15 분 |
| 콤보 시나리오 수식 작성 (비율 0~100% 스윕 로직, 보통 데이터 테이블 또는 별도 행 20개+) | 20-30 분 |
| 슬라이더로 조작 가능한 차트 구성 (Excel Form Control 슬라이더 + 동적 차트, 또는 VBA) | 30-60 분 |
| 크로스오버 지점 찾기 + 결과 해석·리포트 작성 | 15-25 분 |
| **합계** | **90-155 분** |

> 슬라이더 인터랙션은 Excel Form Control + INDEX/MATCH 또는 VBA가 필요해
> 엑셀에 익숙하지 않으면 이 단계에서만 1시간 이상 소요되는 경우가 흔함.

## After (자동)

```bash
./frame run qlc-tlc-bit-cost-calculator --in <params.txt> "<옵션 컨텍스트>"
```

| 작업 | 평균 소요 |
|---|---|
| 파라미터 텍스트 파일 작성 (`key=value`, 샘플 복사 후 값만 교체) | 5-10 분 |
| `frame run` 호출 → calc.py 스윕 + HTML 슬라이더 생성 + report.md 작성 | 10-30 초 (모델 호출 포함) |
| 결과 검토 (report.md 읽기 + HTML 슬라이더 조작으로 감각 확인) | 5-10 분 |
| **합계** | **10-20 분** |

## 무엇이 자동화되지 않는가 (정직)

- **파라미터 자체의 정확도는 사람 책임** — 계산기는 입력된 Step 수/투자비/
  수율 등을 그대로 신뢰합니다. 공정팀·재무팀이 검증한 숫자를 넣지 않으면
  결과도 그만큼만 유효합니다.
- **opex(운영비) 없음** — capex(투자비)만 비교하는 1차 모델. 실제 투자
  의사결정에는 웨이퍼당 운영비를 반영한 재무 모델이 별도로 필요합니다.
- **공정 시뮬레이션 아님** — 레시피 전환 셋업 시간, 실제 가동률, 수율
  램프업 곡선 등은 반영하지 않는 선형 근사 모델입니다.
- **최종 투자 승인은 사람** — 이 계산기는 "1차 스크리닝 + 감각 확인" 용도.
  크로스오버 지점을 찾았다고 바로 투자를 결정하지 않고, 공정/재무팀
  검토를 거쳐야 합니다.
